# 多页作业漏批提示设计

## 问题

多页照片批改时，模型可能只返回部分页面的题目。现有服务仍把结果保存为“批改完成”，结果接口也只返回空的题目数组，导致小程序展示无标注原图，用户容易误认为该页全部正确。

## 设计

批改结果落库前，以实际参与批改的作业图片顺序为准，检查模型返回的 `source_image_index` 是否覆盖每一页。任一页没有题目时，保留已有页面的批改结果，但把整次提交标记为 `needs_review`，并在 `review_reason` 中写明漏批页码。不得把未覆盖页面推断为正确或错误。

结果接口为每页增加 `has_correction` 与 `review_message`。只要该页没有关联题目，接口就返回中性提示“本页未生成批改结果，不能判断为全对，请重新批改或人工复核”。这项动态判断同时覆盖修正上线前已经保存的历史结果。

学生端和家长端复用的整页批改组件在原图紧邻位置显示该提示。已有题目的页面继续显示绿色正确、红色错误和待复核摘要，行为不变。

## 验证

- 两张作业照片仅返回第一页题目时，结果与提交进入 `needs_review`，原因包含第二页。
- 结果接口对无题目页面返回 `has_correction=false` 和明确提示。
- 小程序整页组件渲染该提示，且红绿批注样式不受影响。
- 后端与小程序完整测试保持通过。
