# 家长端与学生端底部导航设计

## 目标

为家长端和学生端增加符合各自任务的底部导航，把家庭设置和身份相关功能集中到“我的”，减少一级页面中的突兀设置入口，并保持任务详情、提交和批改流程专注。

## 方案选择

采用业务页面内的自定义底部导航，不使用微信原生 `tabBar`。原生 `tabBar` 全局固定，无法按家长、学生显示不同栏目，也无法自然处理需要动态计划编号的“学习计划”入口。

自定义导航封装为共享组件，由组件读取当前角色并接收当前栏目。导航使用 `wx.redirectTo` 切换一级页面，避免不断累积页面栈。所有导航标签使用中文。

## 导航结构

### 家长端

- 首页：`pages/parent/home/index`
- 学习计划：打开当前孩子最近一份计划；没有计划时回到首页的导入引导
- 我的：`pages/profile/index/index`

### 学生端

- 今日学习：`pages/student/today/index`
- 我的：`pages/profile/index/index`

“学习计划”所需计划编号由家长首页请求报告时写入本地存储，并在计划生成、确认和打开日历时同步更新。导航组件找不到计划编号时跳转家长首页并显示“还没有学习计划”。

## 显示范围

底部导航只出现在以下一级页面：

- 家长首页
- 家长周期计划
- 学生今日学习
- 我的

导入作业、确认计划、任务详情、专注计时、上传作业、批改结果、报告详情等二级流程不显示底部导航。一级页面统一增加底部安全区留白，避免最后一张卡片被遮挡。

## 视觉与交互

底栏固定在安全区上方，使用白色半透明表面、细绿色边界和轻量阴影。每项包含简洁图形和中文标签；当前栏目使用深绿色，其余栏目使用灰绿色。触摸目标不小于 88rpx。

底栏不覆盖微信原生导航栏，也不改变现有页面标题。切换栏目时不播放复杂动画，只提供按压反馈。无计划等业务错误使用 Toast 和页面空状态，不创建弹窗。

## “我的”页面

共享“我的”页面按 `user.role` 分区展示。

### 家长身份

- 身份与家庭概览
- 当前孩子选择
- 小朋友档案列表
- 添加小朋友
- 家庭邀请码和成员统计
- 加入其他家庭作为次要操作

选择孩子时同时更新 `app.globalData.currentStudent` 和本地存储；返回家长首页或学习计划后立即使用新选择。

### 学生身份

- 当前学生身份与绑定档案
- 所属家庭和家长人数
- 未绑定家庭时显示加入家庭表单
- 已绑定后隐藏家庭邀请码、添加孩子和家长管理功能

学生不能在界面中创建小朋友档案或复制家庭邀请码。

## 页面迁移

- 从学生今日学习页删除“家庭设置”。
- 从家长首页删除“家庭设置”。
- 保留家长首页“导入作业”作为核心业务操作。
- “我的”页成为家庭、孩子和身份设置的唯一一级入口。
- 登录成功后仍按角色进入家长首页或学生今日学习。

## 组件与状态

创建 `components/role-tabbar/`，接收 `active` 属性，允许值为 `home`、`plan`、`study`、`profile`。组件根据 `getApp().globalData.currentRole` 生成对应栏目。

新增导航状态工具，负责角色栏目定义、当前栏目和目标路径计算。工具保持纯函数，以 Node 测试覆盖角色差异和中文标签。

本地存储新增：

- `currentStudentId`
- `currentPlanId`

应用启动和身份上下文加载时恢复当前孩子；存储记录不再有效时回退到接口返回的第一个孩子。

## 错误与空状态

- 家长没有计划：点击“学习计划”返回家长首页并提示“还没有学习计划，请先导入作业”。
- 当前孩子不存在：家长首页和“我的”显示创建孩子引导。
- 学生未加入家庭：学生“我的”显示加入家庭表单，今日学习显示绑定引导。
- 接口失败：保留当前页面，显示中文 Toast，不清空已有上下文。

## 测试与验收

自动化测试覆盖：

- 家长、学生导航栏目和中文标签。
- 当前栏目选中状态。
- 有无计划时的目标路径。
- 当前孩子持久化和无效记录回退。
- “我的”角色可见区块。
- 所有小程序 JavaScript 语法。

微信开发者工具手动验收：

- 家长三个栏目切换。
- 学生两个栏目切换。
- 底栏不遮挡卡片和安全区。
- 家长切换孩子后首页与计划同步。
- 学生看不到添加孩子和家庭邀请码。
- 二级流程没有底栏且返回路径正常。

## 实施顺序

1. 导航状态工具和共享底栏组件。
2. 家长首页、计划页和学生今日学习接入底栏。
3. 当前孩子与当前计划持久化。
4. “我的”按角色重组内容。
5. 移除页面内家庭设置入口。
6. 自动化验证和微信开发者工具验收。
